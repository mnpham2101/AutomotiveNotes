# AutomotiveNotes

## Purpose
This project is a self-study **knowledge base for an Automotive Embedded Software Technical Lead role** (NXP / AUTOSAR Classic / MCAL focus, ~10-engineer team, ASPICE-governed delivery). The content was derived from a specific job description covering: customer-facing communication, technical leadership (code review, estimation, risk), team management, and NXP/MCAL/AUTOSAR technical depth.

Source JD topics map to the knowledge base sections:
1. ASPICE framework and process ownership
2. AUTOSAR architecture (Classic vs Adaptive), MCAL & Driver layer
3. Embedded fundamentals (MCAL/Driver/Bootloader, Cortex-A/M automotive chips) + Linux services/daemons
4. A concrete proposed MCAL application (design, toolchain, build)
5. NXP automotive software platform knowledge
6. ECU development lifecycle
7. Tech-lead tech stack, including a dedicated Yocto build-system deep dive
8. A 20-day phased training plan with checkable progress items

## Layout
```
doc/KnowledgeBase/
  md/                 source markdown, files named "<topic>.<subtopic>-Title.md" (e.g. 2.1-AUTOSAR-Architecture.md)
  md/diagrams/        PlantUML (.puml) sources for embedded diagrams + their rendered .svg output
  tools/               build tooling
    plantuml.jar       vendored PlantUML renderer (Java)
    render_diagrams.py renders md/diagrams/*.puml -> *.svg
    build_site.py      converts md/*.md -> static HTML site under site/, splitting each file's
                        H2 sections into individually navigable pages, with a home page built from
                        0.0-Introduction.md
    requirements.txt   Python deps (markdown, pymdown-extensions)
  site/                generated static HTML output (regenerated by tools, not hand-edited)
  README.md            human-facing build/serve instructions
```

## Rebuilding the site
```
pip install -r doc/KnowledgeBase/tools/requirements.txt
python doc/KnowledgeBase/tools/render_diagrams.py
python doc/KnowledgeBase/tools/build_site.py
python -m http.server --directory doc/KnowledgeBase/site 8000
```
Requires a JRE on PATH (`java`) to render diagrams; `tools/plantuml.jar` is vendored so no other network access is needed at build time, and the generated site has no runtime dependency on the network at all.

## Conventions
- Each markdown file in `md/` is numbered `<topic>.<subtopic>` matching the JD topic list above, and is self-contained: it focuses on one concept and links to sibling files for related context rather than duplicating content.
- Headings are consistent across files where it makes sense: Concept Introduction, Scope, Use Cases, Sample / Architecture Diagram, Q&A, References. The site generator turns every `##` heading into its own navigable HTML page plus a combined full-page view.
- Diagrams are authored as PlantUML source files in `md/diagrams/` and referenced from markdown via standard image syntax pointing at the rendered `.svg`.
- This knowledge base simplifies ASPICE/AUTOSAR/NXP specifics for learning purposes — it is not a certification course or a substitute for official NXP/AUTOSAR documentation (see References sections in each doc).
