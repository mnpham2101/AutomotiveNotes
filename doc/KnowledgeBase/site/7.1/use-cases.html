7.1 Tech Lead Tech Stack Overview

Use Cases

  • Choosing CMake vs. raw Makefiles for a new MCAL project: CMake pays off when the same sources must build under multiple toolchains (S32DS-bundled GCC, a CI Docker image, a different customer's IAR toolchain) — a Tech Lead's tooling decision directly affects long-term maintenance cost, feeding JD 3.4 (technical decision-making).
  • Setting up Jenkins for a mixed MCAL + Yocto project: typically two pipelines — a fast MCAL build+static-analysis pipeline (minutes) and a slower full Yocto image build pipeline (potentially hours), gated differently in the merge workflow.
  • MISRA gate tuning: deciding which MISRA deviations are pre-approved project-wide vs. requiring per-instance justification in code review — a recurring code-review-policy decision for the Tech Lead (JD 3.2).
  • Mentoring: junior engineers often need guided ramp-up on Git workflow and static-analysis-tool literacy before they can contribute productively to review-gated code — direct tie to JD 2.3.
