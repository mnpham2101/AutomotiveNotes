Introduction — Automotive Embedded Technical Lead Knowledge Base

Goals / Purpose

  • Give a structured, fast on-ramp into the concepts, vocabulary, and artifacts a Technical Lead in this kind of role is expected to already know or pick up quickly.
  • Connect four areas that are normally taught separately: process (ASPICE), software architecture (AUTOSAR), platform (NXP silicon + tooling), and leadership (estimation, review, mentoring, customer-facing communication).
  • Provide one worked example (a concrete MCAL application proposal, see 4.1) that ties the abstract concepts to something buildable, with a real toolchain and build script.
  • Provide a 20-day training plan (8.1) so the material can be consumed in a structured, checkable way rather than read once and forgotten.
