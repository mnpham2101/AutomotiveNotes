8.1 20-Day Training Plan

Use Cases — How to Use This Plan

  • Block 60–90 minutes/day; each day's checklist is sized to fit that window including the linked file's Q&A section.
  • Where a day says "drill," it means producing something (a written answer, a diagram redraw, a mock email) — not just re-reading.
  • If short on time, Phase 3 (the hands-on build) is the least skippable — it's the only phase that exercises real toolchain friction, which is where interview/on-the-job credibility is actually tested.
  • Phase 6 is the knowledge-sharing phase: budget extra time (90–120 minutes/day) since each day includes external research, not just reading this knowledge base. Where possible, actually deliver the day's write-up to a colleague rather than filing it away — the point of this phase is practicing the "explain it to the team" skill, not just personal study. Phase 6 days can be reordered or distributed one-per-week as an ongoing team session rather than run back-to-back.
