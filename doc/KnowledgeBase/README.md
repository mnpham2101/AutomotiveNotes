# Automotive Tech Lead Knowledge Base

Source markdown lives in [`md/`](md/), numbered `<topic>.<subtopic>` per the job-description sections this was built from (see [`md/0.0-Introduction.md`](md/0.0-Introduction.md) for the full map). A static HTML site is generated from it under `site/`.

`site/` is **build output, not checked into the repository** (it's listed in `.gitignore`) — run the commands below to generate it locally before serving or browsing it.

## Rebuilding

```bash
pip install -r tools/requirements.txt
python tools/render_diagrams.py   # md/diagrams/*.puml -> *.svg (needs `java` on PATH; tools/plantuml.jar is vendored)
python tools/build_site.py        # md/*.md -> site/ (home page, one page per doc, one page per H2 section)
```

## Serving locally

```bash
python -m http.server --directory site 8000
```

Then open `http://localhost:8000/`. The generated site has no runtime dependency on the network — diagrams are pre-rendered SVGs and styling is a local stylesheet.

## Layout

```
md/                  source markdown (edit here)
md/diagrams/*.puml   PlantUML diagram sources (edit here); *.svg are build output, regenerated by render_diagrams.py
tools/               build scripts + vendored plantuml.jar
site/                generated output (git-ignored — regenerated by build_site.py, do not hand-edit or commit)
```

## Editing content

- Add a new doc: drop a `<topic>.<subtopic>-Title.md` file into `md/`, starting with a single `# Title` line, using `## Heading` for each navigable section. Cross-reference other docs with standard markdown links to their filename, e.g. `[4.1 MCAL Application Design](4.1-MCAL-Application-Design.md)` — `build_site.py` rewrites these automatically into site links.
- Add a diagram: create `md/diagrams/<name>.puml`, reference it from markdown as `![alt](diagrams/<name>.svg)`, then re-run `render_diagrams.py` and `build_site.py`.
- Task checklists (`- [ ] item`) render as checkboxes via the `pymdownx.tasklist` markdown extension.
